# 微信视频号取证平台 — 开发计划

> 版本：v2.0
> 日期：2026-06-22
> 变更说明：聚焦微信视频号平台，快手平台后续再接入

---

## 一、项目背景

### 1.1 当前状态

项目目前 `weixin/` 目录下已有一套完整的微信视频号取证脚本：

| 模块 | 目录 | 平台 | 运行方式 | 核心能力 |
|------|------|------|----------|----------|
| weixin | `weixin/` | 微信视频号 | PC + AScript MCP + ADB | 搜索→录屏→截图OCR→引流检测→博主信息→证据包(JSON+HTML)→ASR转写→剧本比对→MySQL存储 |

**现状特点**：
- 功能较完善但以命令行脚本方式运行，没有统一调度入口
- 依赖 AScript MCP 通信层和 Android 设备
- 取证结果以本地文件形式存储（JSON/HTML/截图/录屏）
- 缺少前端可视化界面，无法远程发起任务和浏览证据
- 不支持多设备同时取证

> **注**：`kuaishou/` 目录下的快手模块功能较基础，本期暂不纳入平台。平台架构保留扩展接口，后续可接入。

### 1.2 目标

在 `platform/` 目录下构建一个**微信视频号取证平台**：

1. **Web 化后端**：将 `weixin/core/` 的采集逻辑封装为可远程调用的后端服务
2. **可视化前端**：提供 Web 界面用于任务管理、实时监控、证据浏览、人工复核、博主聚合分析
3. **多设备支持**：支持单台 PC 同时控制多台手机并行/串行取证
4. **扩展预留**：架构设计上保留多平台接入能力（后续可接入快手、抖音等）

### 2.2 技术选型

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端框架 | **Vue 3 + Element Plus** | 成熟的企业级后台UI组件库，适合专业取证后台风格 |

## 三、数据模型设计

### 3.1 核心实体关系

```
Task (1) ──────── (N) EvidenceRecord
  │                      │
  │                      ├── (N) Screenshot (文件列表)
  │                      ├── (1) MediaInfo
  │                      ├── (1) VideoInfo
  │                      ├── (1) ProfileInfo
  │                      ├── (1) TrafficInfo
  │                      └── (1) ReviewResult
  │
  └── (1) Device

EvidenceRecord (N) ──────── (1) AuthorCluster
```

### 3.2 数据模型（基于现有 weixin EvidenceRecord）

```python
# Task — 取证任务
class Task:
    id: int
    keyword: str                   # 搜索关键词
    status: str                    # pending | preparing | running | completed | failed | stopped
    device_id: str                 # 执行设备标识
    max_videos: int                # 最大采集视频数，0 = 全量
    hold_seconds: int              # 每条视频采集停留时间（默认 240s，用于录屏）
    capture_method: str            # "auto" | "adb" | "scrcpy"
    started_at: datetime
    finished_at: datetime
    created_at: datetime
    log_file_path: str             # 日志文件路径
    error_message: str

# EvidenceRecord — 证据记录（复用 weixin/db/models.py 的 EvidenceRecord）
class EvidenceRecord:
    id: int
    task_id: int
    search_keyword: str
    video_identifier: str          # 稳定唯一标识（sha1）
    fingerprint: str               # 去重指纹（md5）
    capture_timestamp: str
    # 视频信息
    blogger_name: str
    video_channel_id: str          # 视频号ID
    video_channel_id_raw: str
    video_channel_id_needs_review: bool
    title: str
    video_link: str
    publish_time: str
    like_count: str
    comment_count: str
    share_count: str
    favorite_count: str
    # 博主信息
    profile_name: str
    profile_account: str
    subject_type: str              # "企业" | "个人"
    company_full_name: str
    # 引流信息
    has_traffic_marker: bool
    traffic_marker_text: str
    target_blogger_name: str
    target_video_channel_id: str
    target_video_channel_id_raw: str
    target_company_name: str
    target_company_verified_at: str
    # 媒体信息
    recording_video_path: str
    recording_audio_path: str
    recording_duration_seconds: int
    has_audio: bool
    asr_text: str                  # ASR 转写文本
    asr_model: str                 # "sensevoice" | "paraformer" | "xunfei"
    # 剧本比对
    script_match_status: str       # "pending" | "matched" | "not_found" | "error"
    script_match_similarity: float
    script_match_episode: str
    script_match_scene: str
    script_match_script_text: str
    # 证据文件
    json_path: str
    html_path: str
    screenshots_json: str          # 截图文件列表 JSON
    # 复核状态
    review_status: str             # "不确定" (默认) | "侵权" | "白名单"
    reviewer: str
    review_notes: str
    reviewed_at: datetime
    created_at: datetime

# AuthorCluster — 博主聚合（按 video_channel_id 聚类）
class AuthorCluster:
    id: int
    blogger_name: str
    video_channel_id: str
    subject_type: str              # "企业" | "个人"
    company_full_name: str
    total_videos: int
    infringement_count: int        # 侵权数
    whitelist_count: int           # 白名单数
    uncertain_count: int           # 不确定数
    last_capture_time: datetime
    last_updated: datetime

# Device — 设备
class Device:
    id: str                        # 设备唯一标识（ADB serial）
    name: str                      # 显示名称
    status: str                    # online | offline | busy
    ip_address: str                # WiFi IP
    connection_mode: str           # "LocalIP" | "ADB" | "USB"
    screen_width: int
    screen_height: int
    last_checked_at: datetime
```

## 四、取证引擎设计

### 4.1 WeixinCollector — 微信采集器

直接封装 `weixin/core/` 中的现有函数，不重写已验证逻辑：

| 采集步骤 | 封装来源 | 说明 |
|----------|----------|------|
| 设备连接 | `core/main.py` → `connect_device_auto_v2()` | WiFi直连 + ADB扫描 fallback |
| 搜索关键词 | `core/main.py` → `search_keyword()` | 固定坐标 + 分辨率自适应缩放 |
| 点击首个视频 | `core/main.py` → `click_first_video_result()` | 搜索结果页左上角 |
| 播放页确认 | `core/navigator.py` → `wait_for_video_page()` | UI树 + 截图 fallback |
| 采集证据 | `core/collector.py` → `collect_current_video()` | 截图/OCR/引流/博主/链接 |
| 切换下一条 | `core/main.py` → `swipe_up()` | ADB input swipe |
| 证据落盘 | `core/store.py` → `save_record()` | JSON + HTML |
| OCR 识别 | `core/collector.py` → `local_ocr_image()` | PaddleOCR（worker 子进程） |
| 录屏控制 | `core/media_capture.py` | scrcpy / ADB |
| ASR 转写 | `asr/sensevoice.py` / `asr/paraformer.py` | SenseVoice 首选，Paraformer 兜底 |
| 剧本比对 | `asr/script_matcher.py` | 剧本匹配引擎 |

### 4.2 引擎结构

```
engine/
├── weixin_collector.py     # 微信采集流程编排（调用 weixin/core/）
├── device_manager.py       # ADB/AScript MCP 设备连接管理
├── task_scheduler.py       # asyncio 任务队列 + 状态机
└── evidence_builder.py     # 统一证据包构建（JSON + HTML）
```

> **扩展预留**：后续接入其他平台时，可在 `engine/` 下新增 `xxx_collector.py`，遵循相同的接口约定即可。

## 五、API 接口设计

### 5.1 任务管理

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/tasks` | 任务列表（分页、按状态筛选） |
| `POST` | `/api/tasks` | 创建新任务 |
| `GET` | `/api/tasks/{id}` | 任务详情（含状态、进度、日志摘要） |
| `POST` | `/api/tasks/{id}/start` | 启动任务 |
| `POST` | `/api/tasks/{id}/stop` | 停止任务 |
| `POST` | `/api/tasks/{id}/retry` | 重试失败任务 |
| `DELETE` | `/api/tasks/{id}` | 删除任务及关联证据 |

### 5.2 证据查询

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/evidence` | 证据列表（分页、按博主/状态/关键词/时间筛选） |
| `GET` | `/api/evidence/{id}` | 证据详情（含截图列表、录屏路径、ASR结果） |
| `GET` | `/api/evidence/{id}/json` | 原始 JSON 内容 |
| `GET` | `/api/evidence/{id}/html` | HTML 预览内容 |
| `GET` | `/api/files/{path:path}` | 静态文件访问（截图、录屏、HTML 等） |

### 5.3 复核管理

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/reviews/pool` | 待复核池列表（review_status = "不确定"） |
| `PUT` | `/api/reviews/{evidence_id}` | 更新单条复核状态 |
| `POST` | `/api/reviews/batch` | 批量更新复核状态 |

### 5.4 博主聚合

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/authors` | 博主聚合列表（按侵权数排序） |
| `GET` | `/api/authors/{id}` | 博主详情（含统计 + 所有视频列表） |
| `GET` | `/api/authors/{id}/evidence` | 该博主所有证据记录 |

### 5.5 设备管理

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/devices` | 设备列表及实时状态 |
| `POST` | `/api/devices/scan` | 扫描可用设备 |
| `POST` | `/api/devices/{id}/check` | 执行前置检查 |

### 5.6 WebSocket

| 路径 | 说明 |
|------|------|
| `ws://host/ws/tasks/{task_id}` | 实时日志推送（任务执行时每秒推送最新日志行） |

## 九、关键风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| AScript MCP 连接不稳定 | 取证中断 | 自动重连 + 心跳检测 + 连接池 |
| 手机分辨率不一致 | 坐标偏移 | 沿用现有 `scale_point` 自适应缩放 |
| 微信 UI 版本更新 | 控件坐标变化 | 多 fallback（UI树 → OCR → 固定坐标）|
| PaddleOCR 内存泄漏 | 长时间 OOM | 已有 worker 子进程模式，定期重启 |
| 录屏文件过大 | 磁盘占满 | 按任务自动清理 + 空间告警 |
| 多设备 ADB 冲突 | 截图/录屏失败 | 设备级隔离 + 操作互斥锁 |
| 扫码登录 | 取证被阻断 | 前置检查中增加微信登录状态检测 |

## 十、技术注意事项

1. **AScript 导入隔离**：`from ascript.android...` 只能在手机端 AScript 环境运行，后端 Python 不可直接 import。所有手机操作通过 MCP `deploy_and_run` 下发。

2. **代码复用最大化**：不重写 `weixin/core/` 已验证逻辑，`WeixinCollector` 做薄封装层。

3. **OCR 内存管理**：PaddleOCR 在独立子进程（`ocr_worker.py`）运行，已有机制直接复用。

4. **坐标缩放**：沿用 `scale_point` / `scale_rect` 基于 1080×2400 基准自适应。

5. **视频标识符**：沿用 `build_video_identifier()` 算法（搜索词+标题+博主+视频号ID+引流文本+截图hash）。

6. **证据包格式**：维持现有 `EvidenceRecord.to_dict()` JSON 结构，与历史数据兼容。

7. **录屏策略**：默认 scrcpy --record，失败 fallback 到 ADB 截图序列。

8. **ASR 调用时机**：当前代码中 ASR 在全部视频采集完后统一触发（批量处理），平台中保持一致。

## 十一、总结

本计划将 `weixin/` 目录下已有的取证脚本封装为一个完整的 **Web 可视化取证平台**：

- **后端**：FastAPI + asyncio，直接封装 `weixin/core/` 已验证采集逻辑
- **前端**：Vue 3 + Element Plus，专业取证后台风格
- **数据**：SQLAlchemy ORM，兼容 SQLite / MySQL
- **流程**：任务创建 → 前置检查 → 自动采集 → 证据生成 → 人工复核 → 博主聚合

开发分为 **8 个阶段**，预计总工期 **33-46 天**。架构保留多平台扩展接口，后续可接入快手、抖音等平台。
