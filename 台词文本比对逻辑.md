# 剧本台词比对引擎技术文档

## 概述

`script_matcher.py` 是微信视频号侵权取证系统的核心组件，负责将 ASR（语音转文字）输出与剧本台词进行比对，判断视频内容是否使用了受版权保护的剧本。

核心挑战：
- ASR 存在大量**同音/近音字错误**（如"许流年"被识别为"曲流年"）
- ASR 经常**把相邻句子粘连**成一长串
- 单句 ASR（15~40 字）需要与剧本长台词（100+ 字）或短句精确匹配
- 剧本索引规模约 600~800 句，需要高效

**支持的剧本格式：**
- **纯文本格式**：无元数据的纯台词文本，每行一句
- **结构化格式**：含集号（`第3集：xxx`）、场景（`1-1 日 内 地点`）、出场人物、说话人元数据的标准剧本
- **按剧名多剧本**：通过 `SCRIPTS_DIR/{keyword}.txt` 支持多个剧名的独立剧本文件

**最新更新（2026-07）：**
- 新增结构化剧本解析（含集号/场景/人物/地点元数据）
- 新增按剧名多剧本支持
- 新增日志回调到平台前端
- 对话行正则增强（支持中文括号内舞台指示）
- `SequenceMatcher.ratio()` 不再交换参数（已确认为对称方法）

## 第一部分：剧本索引构建

### 1.0 剧本路径解析与格式检测 (`_load_and_parse`)

**路径解析优先级**（`_resolve_script_path`）：

```
1. 显式传入 script_path → 直接使用
2. SCRIPTS_DIR/{keyword}.txt → 按剧名查找
3. SCRIPT_RAW_PATH 环境变量（单文件兜底）
4. 找不到 → 返回 None，不随意用默认剧本（避免误匹配）
```

**格式自动检测**：通过扫描是否含 `第X集：` 行来判断剧本类型：

```python
has_episode = any(RE_EPISODE.match(line) for line in raw_lines if line.strip())
if has_episode:
    _load_structured()   # 结构化格式（有元数据）
else:
    _load_plain_text()   # 纯文本格式（无元数据）
```

### 1.1 结构化剧本解析 (`_load_structured`)

支持含集号、场景、出场人物、说话人等元数据的标准剧本格式。

**示例剧本片段：**

```
第3集：弃子归来
1-1 日 内 顾家别墅客厅
出场人物：顾倾、沈逸风、管家

顾倾（声音嘶哑）流年，二十年了，可你呢？
沈逸风：五百万，买断这个视频的所有权。
▲ 顾倾摔门而去
```

**解析流程：**

```
原始文件
    │
    ├─ 第X集：xxx      → current_episode = "第X集"，进入剧本正文
    ├─ X-X 日/夜 内/外  → current_scene + current_location
    ├─ 出场人物：xxx    → current_characters
    ├─ ▲ 开头的行       → 跳过（舞台指示）
    ├─ "身份标签""外形数据"等 → 跳过（_RE_NON_DIALOG）
    ├─ "角色名（动作）：台词"  → 提取角色名 + 台词文本（_RE_DIALOG）
    │   ├─ 移除括号内舞台指示：`_RE_PAREN_CONTENT.sub("", text)` 
    │   │   "流年，二十年了（抽泣），可你呢？" → "流年，二十年了，可你呢？"
    │   └─ 过滤超长"台词"（>200字，通常是描述行，非口语）
    └─ _add_dialog_line()
```

**对话行匹配正则（增强版）：**

```python
_RE_DIALOG = re.compile(
    r"^([^\uff08：:\n▲]+)"                  # 角色名（不含中文括号、冒号、舞台指示符）
    r"(?:\s*[\uff08(][^\uff09)]*[\uff09)])?"  # 可选括号内舞台指示，如"（愤怒地）"
    r"[:：]\s*(.+)$"                           # 中英文冒号 + 台词文本
)
```

### 1.2 纯文本解析 (`_load_plain_text`)

纯台词文本，每行一句，不含集号/场景/人物等元数据。

**解析流程：**

```
原始文件内容
    │
    ├─ 按换行符拆分
    │
    ├─ 跳过空行
    │
    ├─ 跳过 ▲ 开头的舞台指示（_RE_STAGE_DIRECTION）
    │
    ├─ 跳过"身份标签"、"外形数据"等非台词行（_RE_NON_DIALOG）
    │
    ├─ 按句子切分（_split_into_sentences）
    │   "五百万，买断这个视频的所有权。顺便买断那点可笑的血缘亲情。"
    │   → ["五百万，买断这个视频的所有权。", "顺便买断那点可笑的血缘亲情。"]
    │
    ├─ 过滤：清洗后不足 2 字符的标点碎片丢弃
    │          _add_dialog_line 中再次过滤 ≤4 字符的短句
    │
    └─ 每条句子 → _add_dialog_line()
```

### 1.3 句子切分 (`_split_into_sentences`)

```python
# 按中文句末标点切分，保留标点
parts = re.split(r"([。！？\n]+)", text)

# 将分隔符附加到前一句
# "你好。世界！再见"
# → ["你好。", "世界！", "再见"]
```

切分策略：
- 按 `。` `！` `？` 换行符切分
- **标点归属前一句**，保证语义完整
- 不切逗号（逗号切分在 `match_segmented` 中作为降级策略使用）

### 1.4 对话行构建 (`_add_dialog_line`)

每条句子进入索引前做以下处理：

| 字段 | 来源 | 用途 |
|---|---|---|
| `text` | 原始句子（结构化：去括号舞台指示；纯文本：原句） | 匹配用 |
| `display_text` | 结构化：保留括号的原文；纯文本：同 text | 前端展示 |
| `character` | 结构化：说话人名称；纯文本：空 | 元数据 |
| `episode` | 结构化：`"第X集"`；纯文本：空 | 集号 |
| `scene` | 结构化：场景号；纯文本：空 | 场景 |
| `location` | 结构化：地点；纯文本：空 | 地点描述 |
| `pinyin` | `pypinyin.lazy_pinyin(text)` 空格拼接 | 拼音匹配 |
| `pinyin_syllables` | 拼音音节列表 | 逐音节对比 |
| `clean_text` | 去掉 `，。！？、；：""''（）` 和空白 | 字符级确认 |

**拼音生成示例：**

```
输入: "买断这个视频的所有权"
输出: "mai duan zhe ge shi pin de suo you quan"
```

使用 `pypinyin` 的 `lazy_pinyin` 模式，**不生成声调**（因为 ASR 输出也没有声调信息）。

**过滤规则：**

- 拼音字符 < 2 个 → 丢弃
- 清洗后字符 < 4 个 → 丢弃（避免"你……"、"顾倾。"这种超短句误匹配）

### 1.5 描述行过滤 (`_is_description_line`)

用于结构化剧本，剔除角色描述信息（非口语台词）：

**过滤条件：**

| 标记 | 说明 |
|------|------|
| `"身份标签"`, `"外形数据"`, `"身高："`, `"体重："`, `"形象："` | 角色外观/设定元数据 |
| `"穿着风格"`, `"人设性格"`, `"参考形象"`, `"性格："`, `"简介"` | 人设描述 |
| 台词文本长度 > 200 字符 | 超长行通常是描述文本，非口语台词 |

### 1.6 拼音前缀索引 (`_build_index`)

```python
# 取每条台词拼音的前 4 个字母作为前缀
prefix = pinyin_no_space[:4]

# 索引结构：
{
    "maid": [0, 15, 23],        # 以 "maid" 开头的台词行号列表
    "wu b": [1, 8, 42],         # 以 "wubi" 开头的台词行号列表
    ...
}
```

**注意：** 实际匹配时**已改为全量扫描**而非前缀过滤。因为 `partial_ratio` 模式下，ASR 片段可能从台词中间开始匹配（如"……顺便买断……"），前缀过滤会漏掉正确结果。`_pinyin_to_idx` 字典在代码中保留但未参与实际匹配。全量扫描 600~800 行 + rapidfuzz 的速度足够快（<0.1s/条）。

## 第二部分：ASR 文本分句

### 2.1 入口函数 (`match_segmented`)

ASR 输出的原始文本通常包含 300~600 字，与剧本单句（15~80 字）长度差异巨大，不能直接整段比对。

**4 层分句策略：**

```
ASR 原始文本
    │
    ▼
┌──────────────────────────────────────────────┐
│ 第 1 层：按句末标点切分                           │
│   re.split(r"[。！？\n]+", query_text)          │
│   保留 ≥ 4 字符的片段                              │
│   例如："走吧。不送。好"  →  ["走吧", "不送"]        │
├──────────────────────────────────────────────┤
│ 第 2 层：逗号降级                                 │
│   当句号切分结果 ≤ 2 段时，改为按逗号切：                │
│   re.split(r"[，,]+", query_text)               │
│   保留 ≥ 6 字符的片段                              │
│   例如："走吧，不送，好" → ["走吧", "不送"]           │
├──────────────────────────────────────────────┤
│ 第 3 层：固定窗口强制切分                            │
│   前两层只切出 1 段 → 按 15 字滑动窗口切分               │
│   步长 = 15 - 3 = 12（3 字重叠）                    │
│   保留 ≥ 6 字符的片段                              │
├──────────────────────────────────────────────┤
│ 第 4 层：长句再切分                                │
│   对 > 40 字的单段按逗号再切                         │
│   避免 ASR 两句粘连只匹配到其中一句                    │
└──────────────────────────────────────────────┘
    │
    ▼
  N 个 ASR 片段（每段 6~40 字）
```

**第 3 层滑动窗口示意：**

```
ASR: "买断这个视频的所有权顺便买断你们跟我之间那点可笑的血缘亲情"
     窗口1: "买断这个视频的所有权顺便买"
     窗口2: "    这个视频的所有权顺便买断你们"    ← 12 字后
     窗口3: "        频的所有权顺便买断你们跟我之"
     ...
```

### 2.2 每段独立匹配

每个 ASR 片段调用 `ScriptIndex.match()`，返回最佳匹配结果。

只有综合相似度 ≥ `min_combined_score`（默认 0.50）的片段才计入有效匹配。

## 第三部分：三层匹配算法

### 3.1 核心入口 (`ScriptIndex.match()`)

```python
def match(self, query_text: str, top_n: int = 3, 
          min_pinyin_score: float = 0.40) -> list[dict]:
```

**输入：** 一个经过分句的 ASR 片段（6~40 字）

**输出：** Top-N 匹配结果，每个包含：
- `script_text`: 匹配到的剧本原文
- `similarity_score`: 综合相似度
- `pinyin_score`: 拼音相似度
- `char_score`: 字符级相似度
- `episode`, `scene`, `location`, `character`: 场景元数据
- `matched_window`: 匹配到的剧本窗口文本

### 3.2 第一层：拼音转换

```
ASR 片段 → pypinyin → 拼音串（无声调）
例如：
  "买断这个视频的所有权" → "mai duan zhe ge shi pin de suo you quan"
  "顺便买断那点血缘亲情" → "shun bian mai duan na dian xue yuan qin qing"
```

同步生成 `query_clean`（去标点纯汉字，用字符层级确认）。

### 3.3 第二层：拼音窗口匹配（权重 0.6）

**目标窗口大小：**

```python
target_chars = max(ceil(query_len × 1.1) + 1, query_len + 2)
```

| ASR 长度 | 目标窗口 |
|---|---|
| 10 字 | 13 字 |
| 20 字 | 24 字 |
| 40 字 | 46 字 |

**匹配方式：**

对剧本索引中每一行（600~800 行）：

```
如果行长度 ≤ 目标窗口：
    → 全行拼音 vs ASR 拼音 → rapidfuzz.partial_ratio / 100

如果行长度 > 目标窗口：
    → 滑动窗口提取（步长 = 窗口 // 3）
    → 每个窗口重新生成拼音
    → 取最高 partial_ratio 得分
```

**为什么用 `partial_ratio`？**

`rapidfuzz.partial_ratio` 是"子串匹配"——即使 ASR 串比台词长很多，只要台词拼音是 ASR 拼音的子串，就能得高分。

```
示例：
  ASR:  "maiduanzhegeshipindesuoyouquanshunbian..."
  台词: "maiduanzhegeshipindesuoyouquan"
  → partial_ratio 得分 ≈ 1.0（台词完整出现在 ASR 中）
  → ratio 得分 ≈ 0.4（长度差异拉低）
```

**关键参数：**

| 参数 | 值 | 作用 |
|---|---|---|
| `_LENGTH_RATIO` | 1.1 | 窗口 = ASR 长度 × 1.1 |
| `_STRIDE_DIVISOR` | 3 | 步长 = 窗口 ÷ 3 |
| `min_pinyin_score` | 0.40 | 拼音分低于 0.4 的直接丢弃 |

### 3.4 第三层：字符级确认（权重 0.4）

拼音匹配能解决"同音字"问题，但可能把"势力"误匹配为"示例"。字符级确认解决歧义。

**使用 `difflib.SequenceMatcher`：**

```python
sm = SequenceMatcher(None, query_clean, clean_win)
char_score = sm.ratio()
# SequenceMatcher.ratio() 是对称的，无需交换参数方向
```

> **注意**：旧版代码曾交换参数方向取最大值（当 `len(query) > len(win)` 时），但 `SequenceMatcher.ratio()` 方法是**对称的**（`ratio(a, b) == ratio(b, a)`），双向比较结果相同，已去除冗余逻辑。

### 3.5 综合评分

```python
combined_score = pinyin_score × 0.6 + char_score × 0.4
```

结果按 `combined_score` 降序排列，取 top_n。

## 第四部分：汇总结果结构

`match_segmented()` 返回：

```json
{
    "status": "matched",
    "segments_total": 16,
    "segments_matched": 13,
    "coverage": 0.8125,
    "avg_similarity": 0.805,
    "best_match": {
        "script_text": "...",
        "similarity_score": 1.0,
        ...
    },
    "segments": [
        {
            "asr_segment": "买断这个视频的所有权",
            "script_text": "五百万，买断这个视频的所有权，顺便……买断你们跟我之间那点可笑的血缘亲情。",
            "similarity": 0.896,
            "character": "",
            "episode": "",
            "scene": "",
            "location": ""
        },
        ...
    ]
}
```

| 字段 | 含义 |
|---|---|
| `status` | `"matched"` 当 coverage ≥ 0.15，否则 `"not_found"` |
| `segments_total` | ASR 切分后的总片段数 |
| `segments_matched` | 匹配成功的片段数 |
| `coverage` | 匹配率 = matched / total |
| `avg_similarity` | 匹配片段的平均综合相似度 |
| `best_match` | 单句最佳匹配 |
| `segments` | 逐句匹配详情 |

## 第八部分：可调参数一览

| 参数 | 默认值 | 位置 | 含义 | 调优建议 |
|---|---|---|---|---|
| `_LENGTH_RATIO` | 1.1 | `ScriptIndex` 类常量 | 剧本窗口 = ASR×1.1 | 增大包容更多上下文，减少更精确 |
| `_WEIGHT_PINYIN` | 0.6 | `ScriptIndex` 类常量 | 最终分中拼音权重 | 同音错误多时增大 |
| `_WEIGHT_CHAR` | 0.4 | `ScriptIndex` 类常量 | 最终分中字符权重 | 漏配多时降低此值 |
| `min_pinyin_score` | 0.40 | `match()` / `match_segmented()` | 拼音候选最低分 | 降低→更多候选但更多误匹配 |
| `min_combined_score` | 0.50 | `match_segmented()` | 综合分最低阈值 | 降低→更多片段被标为 matched |
| `_STRIDE_DIVISOR` | 3 | `ScriptIndex` 类常量 | 滑动步长 = 窗口÷3 | 值越大步长越小（更精细但更慢） |
| coverage 阈值 | 0.15 | `match_segmented()` 内部 | matched 判定线 | 降低→更容易判 matched |

from asr.script_matcher import _emit_callback

result = match_segmented(text, keyword="弃子归来震万城")

## 第七部分：使用示例

### 直接匹配（单句）

```python
from asr.script_matcher import match_query, match_segmented

# 单句匹配
results = match_query("买断这个视频的所有权", top_n=3)
for r in results:
    print(f"相似度: {r['similarity_score']:.2%}")
    print(f"台词: {r['script_text']}")
```

### 分段匹配（整段 ASR）

```python
# 整段 ASR 匹配
result = match_segmented(
    "五百万，买断这个视频的所有权，顺便……买断你们跟我之间那点可笑的血缘亲情。钱到账，视频销毁，我以后在顾家就是个透明的尖子生。",
    min_pinyin_score=0.40,
    min_combined_score=0.50,
)

print(f"状态: {result['status']}")
print(f"覆盖率: {result['coverage']:.1%}")
print(f"匹配: {result['segments_matched']}/{result['segments_total']} 段")
print(f"平均相似度: {result['avg_similarity']:.1%}")

for seg in result['segments']:
    print(f"  ASR: {seg['asr_segment'][:40]}")
    print(f"  剧本: {seg['script_text'][:60]}")
    print(f"  相似度: {seg['similarity']:.2%}")
```

## 第九部分：常见问题

### Q: 为什么用拼音而不是直接字符比对？

A: ASR 的同音/近音错误率很高（10~30%），直接比对会大量漏配。拼音比对消除了发音相近但汉字不同的差异，然后字符确认再兜底。

### Q: 什么时候拼音匹配会失败？

A: 当 ASR 识别的发音与原文差异太大时（如漏字、多字、断句位置错误），`partial_ratio` 仍能容忍，但得分会降低。可通过调低 `min_pinyin_score` 来放宽。

### Q: 为什么不用前缀索引加速？

A: 当 ASR 片段从台词中间开始时（如"……顺便买断……"），前缀索引会把正确的台词行过滤掉。全量扫描 600~800 行 + rapidfuzz 的速度足够快（< 0.1s/条）。

### Q: 结构化剧本和纯文本剧本有什么区别？

A: 结构化剧本包含集号、场景、出场人物、说话人等元数据，匹配结果能告诉你是"第3集 顾家别墅客厅 顾倾说的台词"。纯文本剧本只有台词本身。系统会自动检测格式并选择合适的解析器。纯文本中每句话都会切分为独立索引条目，无元数据字段留空。

### Q: 覆盖率 100% 是否意味着完全一致？

A: 不是。覆盖率只表示 ASR 片段中有多少百分比**找到了至少一个**相似度 ≥ 0.5 的剧本匹配。实际相似度看 `avg_similarity` 更准确。
