# 多角色前端 — 一期冻结决策（已对齐 2026-07-17）

> 依据《多角色前端页面需求草案》及联调前拍板。本文档冻结一期范围。

## 流程与权限（已确认）

| 项 | 冻结结论 |
|----|----------|
| 主流程 | 公司提交工单包 → 帮我取证挂链接 → 二阶段取证 → **取证推送公司核查池** → 公司复核 → 取证推送公安（仅侵权） |
| 新建工单 | 上传规范 zip（Excel 剧名 + 同目录 docx 台词），每剧自动创建已提交工单 |
| 帮我取证 | 上传含「剧名 + 侵权视频链接」的 Excel；待取证平台多选（一期仅微信视频号） |
| 一阶段搜链 | **不做**；公司提供链接 |
| 公安派单 | 不做 |
| 推送含义 | 平台内可见；取证端支持**一键推送公司复核**、批量推送公司/公安 |
| 侵权判定 | 二阶段完成后由**公司专人**在核查池/证据详情复核 |
| 白名单/样片 | 一期不考虑 |
| 登录方式 | 角色切换器（localStorage），不做真实鉴权 |
| 调度 | 全人工，不自动派单 |
| 设备管理 | 取证端需独立「设备管理」页 |
| 公安驾驶舱 | 同步取证推公司、公司复核、推公安等操作；含链路统计与最近动态 |

## 一期页面清单

### 公安 `police`
- `/police` 驾驶舱
- `/police/clues` 已推送线索
- `/police/evidence/:id` 只读证据详情

### 公司 `company`
- `/company` 工单列表
- `/company/review-pool` **核查池**
- `/company/work-orders/new` **新建工单**（单 zip 上传 + 打包贴士 + 样板下载）
- `/company/help-collect` **帮我取证**（平台多选 + 链接 Excel）
- `/company/work-orders/:id` 工单详情 + 核查池证据
- `/company/evidence/:id` 证据详情 + **公司复核**

### 取证 `collector`
- `/collector` 调度台（认领、同步链接、跳转链接池）
- `/collector/link-pool` 创建二阶段任务（可带 `work_order_id`）
- `/collector/devices` 设备管理
- 任务 / 证据 / 复核 / 博主等原有页
- 证据列表：**一键推送公司复核** + 批量推送公司/公安（公安仅限公司标侵权）

### 版权 / 文旅
- `/copyright`、`/culture` 占位

### 入口
- `/login` 角色选择
- `/` 按角色重定向
